# Validity Ledger Schema

`evaluation/validity_ledger.json` is generated by `experiments/evaluate.py` for each campaign.

## Fields

- `campaign_id`: campaign identifier from `campaign_meta.json`
- `faithfulness_labels_present`: set of result labels observed in run outputs
- `study_scopes_present`: set of `study_scope` tags observed
- `fault_families_present`: set of fault family tags observed
- `construct_validity`: list of construct-level caveats
- `internal_validity`: list of internal validity caveats
- `external_validity`: list of external validity caveats
- `metric_validity`: list of metric/evaluation caveats
- `paper_faithfulness_policy`: explicit interpretation map for the three labels:
  - `paper-faithful`
  - `paper-faithful-with-proxy`
  - `extended-beyond-paper`

## Purpose

The validity ledger keeps threats-to-validity machine-readable and campaign-specific, so reports and figure annotations can consistently distinguish paper replication claims from extended findings.
