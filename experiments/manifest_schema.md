# Run Manifest Schema

Each run directory generated by `experiments/run_experiments.py` contains `manifest.json` with fields:

- `run_id`: unique ID (`campaign_ts-variant-fault-repeat`)
- `variant`: learner variant name (`paper_fidelity`, `current_default`, ...)
- `fault_id`: canonical taxonomy ID from `fault_matrix.yaml`
- `fault_family`: `paper_aligned` or `extended`
- `study_scope`: `paper_faithful` or `extended`
- `ablation_block`: `fault`, `decision`, `training`, or `som`
- `hypothesis_id`: hypothesis key for grouping in analysis
- `interpretation_note`: plain-language expected effect
- `faithfulness_label`: `paper-faithful`, `paper-faithful-with-proxy`, or `extended-beyond-paper`
- `repeat_idx`: repeat index starting from `1`
- `intensity_level`: ladder label for fault intensity (`low|medium|high|...`)
- `duration_level`: ladder label for injection duration (`short|medium|long|...`)
- `pending_level`: ladder label for evaluator pending window `W`
- `train_mode`: `fresh_bootstrap` or `warm_start`
- `pending_window_s`: prediction pending window `W` used by evaluator
- `durations_s`: object with `warmup`, `bootstrap`, `inject`, `cooldown`
- `fault_on`: command list run through `docker compose run --rm chaos ...`
- `fault_off`: command list for cleanup
- `fault_intensity`: numeric intensity parameter for grouping and analysis
- `learner_env`: full learner runtime config for reproducibility
- `load_profile`: workload generation config used during normal phases
- `started_at`: epoch timestamp
- `campaign_id`: campaign timestamp key
- `matrix_seed`: seed from matrix metadata (for traceability)
- `run_status`: `running|completed|failed`
- `failed_phase`: phase where failure occurred (if any)
- `failure_reason`: captured exception text for failed runs

`run_events.jsonl` complements the manifest with timeline events:

- `type=phase` events for stage transitions
- `type=fault_window` events with `state=on|off`, canonical `fault_id`, `fault_family`, `study_scope`, `ablation_block`, `hypothesis_id`, `faithfulness_label`, `intensity`, `intensity_level`, `duration_level`, and `pending_level`

This schema is designed to keep app-side labels and orchestrator-side labels aligned, especially for toxiproxy network faults.

At campaign root, `campaign_meta.json` records:

- campaign-level provenance (matrix path, mode, generated timestamp)
- git metadata (`git_sha`, `git_branch` when available)
- runtime context (`python_version`, `platform`)
- `run_count` and matrix descriptors.
- load profile metadata (`loadgen_enabled`, profile, rps, work ms)
